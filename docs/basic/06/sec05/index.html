6.5 오만한 정답률의 사기극: 혼동 행렬(Confusion Matrix)의 진실 방

우리가 뉴스 언론 기사에서 흔히 접하는 “의료 인공지능 모델 예측 정확도 99% 달성!” 이라는 카피라이트가 머신러닝 통계학의 관점에서는 얼마나 더럽고 악질적인 깡통 기만 사기 뉴스일 수 있는지 파헤쳐 봅니다. 그 사기성을 타파하고 모델의 진짜 헛발질 치부를 적나라하게 해부하는 혼동 행렬표 4사분면의 냉혹한 세계를 알아봅니다.

6.5.1 성능 평가 지표의 첫 번째 함정 (Accuracy)

초보 데이터 과학자들이 대회에서 점수를 높이려다 가장 먼저 속는 늪이 바로 단순 정확도(Accuracy) 지표입니다. 백분위 퍼센트 단위라서 일반인 사장님에게 보고서로 올리기 아주 좋지만, 실무 현장(Imbalanced 불균형 데이터 환경)에서는 회사를 하루아침에 파산하게 만들 수 있는 치명적 함정입니다.

\[\text{Accuracy} = \frac{\text{모델이 제대로 정답을 찍어 맞춘 데이터 숫자}}{\text{전체 10만 개 데이터 묶음 총합}}\]

6.5.2 깡통 스팸 필터의 대국민 사기극 (Accuracy Paradox)

우리가 매일 사용하는 사내 보안 이메일 분류기를 회귀 머신러닝으로 막 코딩해서 사내 서버에 배포했다고 가정합시다.

[!CAUTION]
정확도 역설 (Accuracy Paradox) 사태 시뮬레이션
오늘 회사 직원들에게 총 10,000통의 메일이 쏟아져 들어왔습니다. 그중 지독한 악성 랜섬웨어 바이러스 [스팸 적군 메일]은 딱 1통뿐이었고, 9,999통은 일반 팀원들이 보낸 정상 메일이었습니다! (현실의 전형적인 ‘정상’ 불균형 데이터 뻥튀기 환경)

이때, 학습 과정에서 너무 미분 역전파가 귀찮았던 쓰레기 파업 AI 모델봇이 분석이고 뭐고 전부 다 포기하고 그냥 “무조건, 나한테 들어오는 이 세상 모든 메일은 100% 정상 폴더 창구로 박아버려 패스!!” 라고 땡깡을 부리며 일괄 처리해 버렸습니다.

$\to$ 자, 이 무식한 파업 AI봇의 최종 점수 평가표 엑셀 결과를 봅시다!

  • 10,000통 분모 중에 정상 메일 9,999개를 어쨌든 결과론적으로 ‘정상’이라고 똑똑하게(?) 다 만점으로 아주 잘 맞췄습니다! 정답 +9,999점!
  • 근데 사내 서버를 폭파시킬 핵폭탄 랜섬웨어 1개만 그냥 ‘정상’이라고 멍청하게 통과시켜서 오판으로 틀렸습니다! (감점 -1점)
  • $\to$ 공식 수학 모델의 [최종 정확도(Accuracy)] 평가지 점수는 무려 99.99% (9999/10000) 로 도출되며 글로벌 해커톤 대회 1등 만점짜리 갓 모델이라고 대서특필됩니다!

이처럼 단순 정확도 비율 하나만 맹신했다가는, 진짜 우리가 목숨을 걸고 필사적으로 그물망으로 잡아내려고 했던 그 단 1개의 치명적인 범인(핵폭탄 랜섬웨어)을 기계가 눈앞에서 뻔히 패스시키고도 밖에서는 정확도 만점이라고 가만히 박수를 치는 끔찍한 참사가 발생합니다.

6.5.3 진실의 방: 혼동 행렬 (Confusion Matrix) 검문소

위와 같은 단순 정확도 99% 의 사기 행위를 원천봉쇄하고 모델의 비리를 박살 내기 위해, 통계학자들은 4칸짜리 진실의 십자가 방(크로스오버 채점 엑셀표) 으로 AI를 머리채 잡고 끌고 들어가서 앉힙니다.

이 방에서는 모델이 단지 정답을 맞혔냐 틀렸냐 1차원적인 채점 비율을 벗어나, “네 놈이 통과시킨 범죄의 죄질이 도대체 어떻게 억울하게 틀린 방향이었고, 어느 구멍으로 눈 뜬 장님 헛발질을 치며 속아 넘어갔는지” 모델의 예측 예측 성향 치부를 4개의 케이스로 적나라하게 감별하고 팩트 체크합니다.

6.5.4 혼동 행렬 4대장 용어 완벽 해부 (스팸 경찰 비유)

이 4가지 수식 T(True)/F(False) - P(Positive)/N(Negative) 용어 기호들은 평생 AI 개발자를 헷갈리게 괴롭히지만, 한 번의 완벽한 역할극 비유로 뇌파에 체화시켜 버릴 수 있습니다.

  • P (Positive - 양성) : 기계 모델이 스스로 사이렌을 켜고 “이놈 내가 잡던 타겟 범인(스팸/테러범) 맞습니다!” 라고 강하게 무전기로 수갑을 채우며 주장/예측함!
  • N (Negative - 음성) : 기계 모델이 무전기를 내리고 한숨 돌리면서 “이놈은 제가 잡던 타겟이 아니고 안심 일반 시민이네요. 통과!” 라고 무사 방관함.
  • T(True) 와 F(False) : 위에서 기계 경찰관 놈이 무전기로 외쳤던 예측 결정이, 나중에 뚜껑 열고 현장 확인해 보니까 진짜 100% 정답(T)이었냐? 기계 혼자 설레발치던 멍청한 오판(F)이었냐?
감식표 코드 AI 예측 무전기 나중에 까본 현실 진실 치명적 죄질 결과(파문) 해설
TP
(특진)		 “범인 잡았다!!”(P) 진짜 악당 맞음 (T) (경찰 포상급 대성공) 모델이 진짜 거대한 스팸 랜섬웨어를 기가 막히게 잘 냄새를 맡고 잡아내서 휴지통에 쳐박음. 아주 이상적!
TN
(평시)		 “정상 시민임”(N) 진짜 시민 맞음 (T) (일상 성공) 모델이 일반 구글 결제 메일이나 어머니 안부 메일을 검문소에서 얌전하게 내버려 둬서 수신함에 정상 도착함.
FP
(1종 오류)		 “범인 잡았다!!”(P) 사실 착한 대학생(F) (억울함 대폭발-조작 수사 에러) 모델이 억지로 무고한 스탯을 엮어서 선량한 일반인 대학생한테 범인 수갑을 채워(P) 강제로 감옥에 가둬버림!

$\to$ 치명타: 스팸 필터 오판으로 사장님이 보낸 10억짜리 중요 사업 계약 메일이 스팸 휴지통으로 삭제 폭파되어 날아가는 최악의 참사 발생!!
FN
(2종 오류)		 “정상 시민임”(N) 진짜 폭탄 테러범(F) (경찰청장 사퇴급-맹인 헛발질 에러) 모델 경찰관이 순진하게 뒷돈 먹고 속아서(N) 주머니에 원자 폭탄을 든 진짜 스팸 테러범을 굽신거리며 공항에 무사 통과시켜 줌!

$\to$ 치명타: 지가 혼자 정상이라고 속고 프리패스로 들여보내 온 랜섬웨어 메일 클릭 한 번에, 사내망 은행 완전 개털리고 해커한테 은행 잔고 코인 100억 증발해킹당함!! 무능의 극치!

이처럼 살 떨리는 진실의 혼동 행렬표를 만들고 나면, 인공지능이 무식하게 저지른 씻을 수 없는 범죄(특히 FP 와 FN 둘 중 하나의 치명타)를 우리 두 눈으로 수학적으로 똑똑히 감찰 집계할 수 있습니다.

바로 다음 수업에서, 우리는 이 지표 표에 수술용 메스를 들이대어 “억울한 놈 1명을 빵에 넣더라도 범인을 절반만 놓치고 다 잡을 것인가?” 라는 철학적 파라미터 계측 곡선들(정밀도, 재현율 평형)을 설계해냅니다.
