통계의 종말: 투사층 룩업 테이블 트릭 (NNLM)

아무리 N-gram 마르코프 체인 타협안을 써도 문장이 조금만 길어지면 여전히 카운터 에러 0원 분모를 뿜으며 터지던 구시대 통계를 완전히 은퇴시킨 전설의 모델입니다. 언어 모델 역사상 최초로 단어 빈도수(Count) 부서의 해체를 선언하고, 뇌세포 신경망 장부를 도입한 피드포워드 신경망 언어 모델(NNLM)의 위대한 혁신과 한계를 뜯어봅니다.

00. 구시대 N-gram 통계 모델의 영구적인 한계 (OOV 붕괴)

우리는 4주차에서 N-gram 통계 방식을 배웠습니다. “저 소년은 참( )” 이라는 문맥 단어가 주어졌을 때, 다음에 나올 단어를 맞추기 위해 구글 데이터베이스를 다 뒤져서 Count 분수 나누기를 시전했습니다.

하지만 기계가 오늘 처음 보는 신조어 단어, 혹은 처음 보는 희소한 단어 조합이 입력으로 뚝 떨어지면 어떻게 될까요? 카운터가 여전히 0(Zero Division, Sparsity 에러)이 되면서 확률을 뿜어내지 못하고 랙이 걸리며 붕괴합니다. 이 희소성의 굴레에서 통계학은 절대 벗어날 수 없었습니다.

01. 최초의 혁명: 뉴럴 네트워크 언어모델 (NNLM)

NNLM은 기계 학습의 파괴적 혁신을 불러옵니다. “야!! 단어의 등장 횟수 따위 숫자 장부 적는 카운팅 짓거리 당장 집어치워!! 우리는 이제 인공 신경망(Neural Net)이 미적분과 스탯 가중치를 조절해가면서 앞 단어를 보고 다음 단어를 유추 추론하게 만들 거다!” 라고 도발하며 등장했습니다.

  • N-gram 꼼수처럼 과거의 모든 문장을 보는 건 포기하고 시야를 줄여 최근 $n$개의 단어만 들여다봅니다.
  • 하지만 통계 수치 분수를 쓰는 게 아니라, 여러 단어를 은닉망(Hidden Layer) 가중치 $W$ 구조로 복잡하게 연결 계산하여 다음 단어를 추론합니다.

02. NNLM의 핵심 구조: 투사층 (Projection Layer) 은닉 공간

NNLM은 아주 독특하게 신경망 구조 첫 번째 관문으로 투사층(Projection Layer) 이라는 커다란 껍데기 공간을 가집니다. 일반 신경망의 뇌세포 활성화 함수(시그모이드 같은 것)가 전혀 없는 텅 빈 고속도로 통과 레이어입니다.

10만 칸짜리 거대하고 쓸모없는 0 이 가득한 원-핫 벡터 배열이 이 투사층 고속도로 문을 통과하면, 엄청나게 커다란 가중치 행렬 $W$와 곱해집니다. 이 수식 과정을 거치고 나면 10만 칸짜리의 쓰레기 행렬 벡터가 $\to$ 아주 콤팩트한 128차원 정도의 꽉 찬 실수 임베딩 우주 좌표(Dense Vector) 공간으로 쪼그라들면서 아름답게 체중 감량(압축)을 합니다!

03. 천재적인 트릭: 투사층 임베딩 룩업 (Lookup Table)

대충 저 설명을 보면 컴퓨터가 거대한 100,000칸 배열 과 가중치 행렬 $W_{100000 \times 128}$ 을 수백만 번 무식하게 고통스럽게 다 일일이 행렬 곱셈 수학 연산을 해야만 할 것 같습니다. 엄청난 그래픽 카드 GPU 전기세가 폭파할 것 같습니다. 하지만 그들은 곱셈을 하지 않는 치명적인 트릭을 심어놨습니다.

[!IMPORTANT]
📖 초심자를 위한 쉬운 해설: 곱하기의 진실 (사실 곱하는 척만 함)
벡터 데이터가 [0, 0, 1, 0, 0] 처럼 3번째 칸에 단 한 개의 1만 켜져 있고 나머지 모두가 곱해봐야 0이 나오는 원-핫 매트릭스라면?
아까 말한 그 엄청난 행렬 곱셈식의 최종 수학적 결과물 답안지는, 사실 존나 큰 가중치 테이블 행렬 $W$에서 3번째 줄(Row)의 실수 숫자값 리스트를 쭉 긁어서 복사하기(Copy) 해오는 것과 결과가 소름 돋게 100% 똑같이 떨어집니다!

따라서 NNLM 엔지니어들은 값비싼 GPU 전력을 낭비하며 미친 곱하기 수식을 시전하지 않습니다!! 그냥 도서관에서 책 꺼내듯이(Lookup) “어? 인덱스 3번이네? 3번째 줄 데이터 꺼내와!” 라며 무식하게 엑셀 색인 검색(Look-Up)을 해서 값을 빼옵니다. 이 혁신적인 스킬 덕분에 연산 속도가 수백 달러어치 비약하게 됩니다.

04. NNLM의 최종 결론과 승리 사유

통계를 찢어버린 위대한 한 걸음이었지만, 여전히 치명적인 결함이 있었습니다.

  • 승리한 이유: 카운트 횟수가 0이라 터지던 기존 알고리즘과 달리, 내부에 가중치 행렬이라는 ‘조정자’를 품고 있기 때문에 생판 모르는 단어 시퀀스가 나타나더라도 “어? 아까 그 비슷한 단어 임베딩 주변 좌표에 있던 녀석이니까 얘도 이 확률이겠다!” 라며 눈치껏 유사도를 유연하게 방어(일반화)하는 대성공을 거두었습니다!
  • 한계점 극복 실패: NNLM은 뇌세포인 은닉층(Hidden Layer)의 가중치 미분 역전파 연산을 여전히 무겁게 들고 있었습니다. 그래서 딥러닝 망이 너무 깊어져서 수백만 개의 문장이 들어가면 속도가 거북이처럼 질질 끌려 한 세월이 걸렸습니다.

이 무거운 병렬 은닉층 병목 현상을 쳐다보다 빡친 글로벌 공룡 그룹 구글(Google)의 천재 군단들이, 이 신경망을 다이어트시키기 위한 칼을 빼 들게 되니, 바로 다음 챕터에 등장할 제왕 Word2Vec 알고리즘 모델의 탄생입니다.
