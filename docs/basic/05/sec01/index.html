차원의 저주와 임베딩(Embedding)의 좌표 마법

통계의 시대를 넘어 인공신경망의 시대가 도래했습니다. 희소한(Sparse) 원-핫 1차원 배열에 갇혀 완전히 남남으로 지내던 단어들을, 의미와 맥락이 숨 쉬는 ‘기하학적 밀집 차원 공간(Dense Vector)’으로 쏘아 올리는 황홀한 워드 임베딩(Word Embedding)의 사상을 배웁니다.

00. 원-핫 인코딩(One-hot)의 복습과 절망

딥러닝 이전 시대(3주차)에 배웠던 원-핫 인코딩의 치명적 한계를 다시 떠올려 봅니다. 원-핫 인코딩은 내 자리 인덱스에만 전구 1을 켜고, 나머지 온 동네 차원 축에는 무자비하게 0을 다 때려 박았습니다.

\[\vec{v}_{\text{사과}} = [0, 0, 1, 0, 0, \dots, 0] \in \mathbb{R}^{|100,000|}\]

01. 치명적 약점 1: 차원의 저주 (Curse of Dimensionality)

사전에 등록된 영단어가 10만 개라면, 고작 “사과”라는 단어 스펠링 한 개를 컴퓨터 메모리 RAM 에 올리기 위해 무려 99,999칸의 쓸데없는 빈칸(0, 암흑 물질)을 할당해서 그려 넣어야 합니다!

이처럼 배열의 칸 수는 무한히 수십만 칸으로 늘어나는데, 정작 의미 있는 알맹이는 단 한 칸인 거대한 비율의 불일치 낭비 현상을 통계학에서 [차원의 저주] 라고 부릅니다. 이 희소 행렬(Sparse Matrix) 텅 빈 상태로 딥러닝 연산을 돌리면 컴퓨터 그래픽 카드(GPU)는 단 몇 분 만에 용량이 터져 폭발합니다.

02. 치명적 약점 2: 상호 직교 (유사도 도출 완전 불가)

더 무서운 물리적 문제는, 원-핫 벡터들이 공간에서 무조건 서로 90도 수직(Orthogonal)으로 놓인다는 점입니다.

  • 우리가 아는 ‘개’ 와 ‘강아지’ 는 사실상 거의 똑같은 동의어입니다.
  • 하지만 원-핫 컴퓨터 뇌 구조에서는 두 벡터의 코사인 각도 내적 연산 결과가 잔인하게도 0 (완전 수직, 무관함) 입니다.
\[\vec{v}_{\text{개}} \cdot \vec{v}_{\text{강아지}} = [1, 0, 0] \cdot [0, 1, 0] = 1\times0 + 0\times1 + 0\times0 = 0\]

컴퓨터 입장에서는 “강아지와 개 사이의 거리나, 강아지와 핵폭탄 사이의 거리나 사실 기하학적으로 100% 똑같이 남남이야!” 라고 판언해 버리니 의미 분석망 생태계가 파괴됩니다.

03. 구세주의 등장: 워드 임베딩 (Word Embedding)

위 두 가지 치명적 단점을 완벽히 박살 낸 딥러닝 인공지능의 최고 축복 시스템입니다.

“무식하게 10만 칸으로 쫙 벌려 놓은 차원(원-핫)을 뭉개고 압축해서, 성별/크기/온도 등 단어의 속성을 담은 256차원짜리 꽉 들어찬 [밀집 벡터(Dense Vector)] 상자 안에 눌러 담자!”

  • 0, 1 같은 정수 딱지가 아니라, [-1.24, 0.88, 3.14] 처럼 소수점 끝까지 밀도가 차 있는 실수(Float) 좌표 평면 배열을 가집니다.
  • 드디어 단어들 사이에 거미줄 같은 위상 각도가 생기며 기하학적 유사도 연산이 인간 수준으로 섬세하게 가능해집니다.

04. 워드 임베딩 유사도 3D 우주 연산

임베딩 압축을 통해 10만 차원의 거대한 우주를 우리가 볼 수 있는 3D 큐브 공간(X, Y, Z)으로 축소 압축 우겨 넣었다고 상상해 봅시다. 단어들이 우주 속의 행성들처럼 공간의 실수 좌표 어딘가에 점으로 박혀서 둥둥 떠다닙니다!

공간에 실수 좌표점이 확실히 찍혔으므로, 이제 인공지능은 마침내 자(Ruler)를 들고 점과 점 사이의 물리적 거리를 잴 수 있습니다.

  • 자로 재어보니 woman 행성과 man 행성 사이의 거리가, princess 단어행성 보다 훨씬 더 가깝습니다! 즉 컴퓨터가 마침내 “이 두 단어는 서로 의미적으로 가족처럼 유사하구나!” 라고 깨닫게 됩니다.

05. 전설적인 워드 임베딩 추론 방정식 (King - Man = ?)

워드 임베딩 알고리즘의 천재적인 성공을 전 세계에 알린 전설적인 수학 방정식 단어 덧뺄셈 연산입니다. 알파벳 스펠링 덩어리들을 좌표 공간의 실수 벡터로 만들어버렸으니 기하학적인 덧셈 연산 수식을 돌려봅니다.

\[\vec{v}_{\text{King}} - \vec{v}_{\text{Man}} + \vec{v}_{\text{Woman}} \approx \vec{v}_{\text{Queen}}\]

[!TIP]
📖 초심자를 위한 쉬운 해설: 딥러닝이 여왕을 찾는 법
딥러닝 AI 모델이 무슨 사람처럼 [왕]과 [여왕]이 부부 관계라는 성별 뜻을 국어사전처럼 공부하고 이해해서 아는 게 절대 아닙니다!

컴퓨터는 그저 왕(King)의 실수 좌표계 위치에서 성질 축 방향(남자의 특성 Vector) 만큼 뒤로 쭈욱 스윽 빼버리고 수직을 그어봅니다. 그리고 그 위치에서 다시 성질 방향 축(여자의 특성 Vector) 만큼의 각도 길이를 쓱 앞으로 더해갔습니다.

그랬더니 그 연산의 종착지 허공 3D 우주 공간에, 우연히 아주 근접한 실수 좌표 거리 내에 ‘Queen’ 이라는 단어 별이 둥둥 떠다니고 찍혀있을 뿐입니다. 이것이 바로 임베딩의 기하학적 매핑 마법입니다!
