# 5.2 통계 카운트의 종말: 투사층 룩업 테이블 트릭과 NNLM

아무리 N-gram 마르코프 체인 타협안을 써도 문서 문맥이 조금만 길어지면 여전히 카운터 에러(분모 `0`)를 뿜으며 터지던 구시대 통계를 완전히 역사의 뒤안길로 날려버린 전설적인 구조 모델입니다. 단어 빈도수(Count) 부서의 해체를 선언하고, 인공의 뇌세포 신경망(Neural Network) 장부를 도입한 최초의 언어 모델 **NNLM (피드포워드 신경망 언어 모델)** 의 위대한 혁신과 병목 투사층 한계를 뜯어봅니다.

---

## 5.2.1 구시대 통계 모델의 영구적인 한계 (OOV 붕괴)

우리는 바로 앞 4주 차에서 빈도수 카운트의 맹점을 뼈저리게 배웠습니다. 
**"저 소년은 참 ( )"** 이라는 문맥 단어가 주어졌을 때 다음에 나올 단어를 알아내려면, 구글 데이터베이스를 다 뒤져서 일치하는 과거의 통계치 결괏값 $\text{Count}$ 로 분수 쪼개기 수식을 시전했습니다.

하지만 기계가 평생 살아오며 단 한 번도 본 적이 없는 신조어 조합이나 희소한 지명 텍스트 코드가 입력으로 뚝 떨어지면 어떻게 될까요? 카운터가 여전히 0(Zero Division, Sparsity 에러)이 되면서 단어 확률 곡선을 뿜어내지 못하고 서버 랙이 걸리며 다운됩니다. **이 희소성의 굴레에서 단순 통계 엑셀학은 결국 영원히 벗어나지 못했습니다.**

---

## 5.2.2 최초의 혁명 신경망: NNLM (Neural Net Language Model)

이에 NNLM 시스템은 기존 기계 학습의 판도를 완전히 뒤집는 패러다임 선언을 단행합니다.
**"야!! 단어가 문서에서 몇 번 나왔는지 그깟 횟수 카운팅 하는 데이터 장부 기록 짓거리 당장 갖다 버려!! 우리는 이제 인공 신경망(Neural Net)이 딥러닝 가중치($W$, $b$) 미적분을 조절하면서 다음 단어를 인간처럼 감으로 추론하게 훈련시킬 거다!"** 

![NNLM Architecture Overview](./img/nnlm_architecture.svg)

NNLM은 복잡한 4개의 계층(Layer)으로 뇌 구조를 설계했습니다.
*   **1. 입력층 (Input Layer)**: 과거 단어들의 엄청나게 긴 10만 차원(차원의 저주) 짜리 원-핫 뼈대 벡터들이 들어오는 대문.
*   **2. 투사층 (Projection Layer)**: 활성 함수(시그모이드 등)가 존재하지 않는 텅 빈 껍데기 선형 결합 우주 공간.
*   **3. 은닉층 (Hidden Layer)**: 엄청난 두께를 가진 가중치 미분 역전파 연산망 (GPU 파괴의 주범 공간).
*   **4. 출력층 (Output Softmax)**: 다음 단어 1개를 고르기 위해 10만 개 단어의 확률 그래프를 토해내는 종착역.

방식은 N-gram 꼼수(마르코프 타협)처럼 최근 $N$개의 직전 단어 시야만 쳐다본다는 점에서는 옛날 모델과 똑같습니다. 하지만 **수학적 내부 통계 수치 분수를 쓰는 게 아니라, 여러 단어 덩어리를 은닉망(Hidden)의 가중치 행렬로 뒤섞어서 스스로 정답을 찾아가도록 진화**했다는 점이 위대한 발걸음입니다.

---

## 5.2.3 뇌 구조의 핵심: 투사층 (Projection Layer) 의 고속도로 압축

NNLM의 첫 번째 관문인 **투사층 (Projection Layer)** 은 딥러닝 역사상 가장 독특한 껍데기 공간입니다. 일반 신경망의 뇌세포 꼬임 활성화 스위치(비선형 함수) 계단이 전혀 존재하지 않는 텅 빈 선형 하이웨이(Highway) 고속도로입니다.

10만 칸짜리 거대하고 쓸모없는 `0` 전구만 잔뜩 켜진 원-핫 벡터 배열 무식쟁이들이 이 투사층 대문을 일단 통과하고 나면, 아주 거대한 가중치 행렬 세트 $W$와 마주치며 강제 곱셈 역결합을 당합니다. 
놀랍게도 이 수학적 수술 고문을 거치면 10만 칸짜리의 쓰레기 행렬 벡터 공간 좌표가 $\to$ 아주 작고 콤팩트한 **128차원 정도의 꽉 찬 실수 우주 좌표(Dense 임베딩 벡터)** 공간으로 예쁘게 쪼그라들면서 기하학 매핑(Mapping)이라는 압도적인 다이어트에 성공합니다!

---

## 5.2.4 천재 엔지니어들의 트릭: 룩업 테이블 (Lookup Table) 스킬

위의 이론 설명만 들으면, 컴퓨터 그래픽 카드가 미친 듯한 GPU의 피를 토할 것 같습니다. 
`[100,000칸 배열]` 과 `[가중치 행렬 $W$ (100000 x 128 크기)]` 끼리 수백만 번의 반복적인 무식한 선형대수 행렬 곱셈을 해야 한다니, 상상만 해도 전기세가 폭발합니다. 하지만 구글/스탠포드 등의 엔지니어들은 이 대규모 곱셈을 굳이 하지 않는 치명적인 꼼수를 심어놨습니다.

![Lookup Table Trick Mechanism](./img/lookup_table_trick.svg)

> [!IMPORTANT]  
> **📖 초심자를 위한 쉬운 해설: 룩업 테이블은 곧 "복사 붙여넣기(Copy & Paste)" 다**  
> 입력 벡터 데이터가 `[0, 0, 1, 0, 0]` 처럼 3번째 인덱스 자리에만 전구 `1` 이 딱 하나 켜져 있고, 그 앞뒤의 나머지 99,999칸은 싹 다 `0` 이 켜진 극단적인 원-핫 배열이라고 곰곰이 상상해 보십시오.  
> 
> 이 거대한 배열과 $W$ 행렬을 아무리 각 잡고 미친 듯이 일일이 다 수식 곱하기 연산을 돌려봤자, 결국 어차피 `0`이 곱해진 애들은 싸그리 쓰레기로 사멸해 날아가 버립니다! 최종적으로 딱 떨어지는 답안지 결괏값은 수학적으로 무조건 **"거대 가중치 행렬 $W$ 테이블에서 3번째 줄(Row)의 실수 리스트를 싹 다 긁어 복사해온 것"**과 두 값이 소름 돋게 100% 똑같이 일치하게 됩니다!   
> 
> *   **엔지니어의 편법**: "야! 어차피 $W$ 행렬 몇 번째 줄인지 짚어내기만 하면 정답이랑 똑같잖아? 값비싼 GPU 전력을 낭비하며 미친 곱하기 수식을 시전하지 마!! 그냥 책장에서 책 꺼내듯이 색인(Lookup)해서 **'어? 원-핫 3번에 전구 불 들어왔네? 그러면 계산 생략하고 걍 뒤도 보지 말고 $W$테이블 3번째 줄 데이터 꺼내서 복붙(Ctrl+C, Ctrl+V) 해와!'**" 
>
> 이 혁신적인 우회 도로(Lookup) 스킬 덕분에 연산 속도는 수천 시간 어치나 비약적으로 가속됩니다.

---

## 5.2.5 NNLM의 최종 결론: 한계 돌파 실패 사유

통계학 공식을 박살 낸 딥러닝(Neural Net) 언어 모델의 위대한 첫걸음이었지만, 여전히 이 NNLM은 치명적인 연산 속도 결함을 안고 무너졌습니다.

*   **🏆 카운트 에러(Sparsity) 방어의 승리**: 카운트 횟수가 0이라 오류를 뱉던 과거 알고리즘과 달리, 내부에 가중치 행렬($W$)이라는 똑똑한 스탯 '조정자'를 품었습니다. 생판 모르는 특이한 단어 시퀀스가 입력되더라도 기계는 *"어차피 이거 아까 훈련했던 비슷한 단어들과 임베딩 3D 우주 좌표 위치가 구역이 거의 겹치네(가깝네)? 그럼 얘도 대충 문맥 확률이 똑같다고 치자!"* 라며 눈치껏 유사도를 유연하게 넘겨버리는 대성공을 거두었습니다!
*   **💣 은닉층(Hidden Layer) 비대증 병목 해결 실패**: 피드포워드 신경망 구조상, 뇌세포 집합인 **은닉층의 거대한 가중치 활성화 함수(시그모이드, Tanh 등)를 거치는 미분 역전파 연산망**을 여전히 너무 무겁게 들고 있었습니다. 그래서 딥러닝 텍스트망 훈련이 시작되면 연산 병렬 처리가 체했다시피 걸려서 속도가 거북이처럼 질질 끌리는 재앙을 겪습니다.

이 무거운 비만 은닉층 뇌세포의 끔찍한 병목 현상을 쳐다보다가 제대로 열이 뻗친 초거대 빅테크 **`구글(Google)`** 의 천재 악마 군단들이, 신경망 뇌 구조를 절반으로 쪼개서 수술해 버리는 폭력적인 칼잡이 다이어트를 단행하게 됩니다. 이것이 전 세계를 공포로 몰아넣은 자연어 임베딩 영원한 대장 **[Word2Vec (구글 워드 임베딩)]** 전설의 서막입니다.
