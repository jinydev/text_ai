# 2주차. 텍스트 전처리와 통계적 접근

2주차 과정에서는 기계가 텍스트를 인식하기 위해 전처리하는 과정과 쪼개는 방법을 알아봅니다.

* **[섹션 01. 텍스트 데이터의 언어학적 계층 구조](/basic/02/sec01/)**
  * 컴퓨터가 고기를 먹는 토큰화 비유
  * 텍스트 분석 스펙트럼 (어휘/구문/의미 표현)
* **[섹션 02. 토큰화(Tokenization) 기술과 품사 태깅](/basic/02/sec02/)**
  * 토큰 단위 크기 논쟁 (문자 vs 단어 vs 문장)
  * 품사 태깅(POS)의 역사: 꽉 막힌 사전 vs 확률적 눈치
* **[섹션 03. 문맥의 보존 기술: N-gram과 RAG 청킹](/basic/02/sec03/)**
  * 구 표현의 혁명: N-gram 기차 칸 매달기
  * 텍스트 덩어리화(Chunking)와 알파벳 BIO 태그
  * 현대 RAG 검색 증강 시스템의 청킹 기술 딜레마
* **[섹션 04. 전처리 지옥의 6단계 파이프라인과 정제](/basic/02/sec04/)**
  * 데이터 청소(Preprocessing) 막노동 6단계 조립 라인
  * 쓰레기 필터 소각장: 정제(Cleaning)의 실체
  * 가족 호적 강제 통합: 정규화(Normalization)
* **[섹션 05. 마법의 그물망 정규표현식과 형태소 다이어트](/basic/02/sec05/)**
  * 문자열 검색 제왕: 정규표현식(Regex)
  * 다이어트 성형: 표제어 추출과 어간 추출 (어원 찾기)
  * 한글 전처리: 교착어 조사 분해기 (KoNLPy)
* **[섹션 06. LLM의 미지 언어 방어막: 서브워드 토큰화 (BPE)](/basic/02/sec06/)**
  * 고전 NLP 심장마비의 원인: OOV 에러
  * 띄어쓰기의 배신과 알파벳 쪼개기(BPE)의 구원
