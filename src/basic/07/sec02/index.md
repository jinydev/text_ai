# 잠재축의 파괴적 차원 압축: 잠재의미분석(LSA)

단순히 문서들을 물리적 거리로 묶는 1차원적 수준을 넘어, 무식하게 거대한 엑셀 단어 행렬(DTM)의 차원을 프레스기처럼 강제로 찍어 누르며 선형 대수학의 기하학적 매트릭스 기술로 숨은 토픽 축을 캐내는 LSA 기법을 만납니다.

---

## 00. 차원의 저주를 빗겨가는 특이값 분해 (SVD)
과거 3주차에 우리는 DTM(문서-단어 행렬) 엑셀표를 만들었습니다. 
세로 1,000만 줄짜리 코퍼스(문서)에 가로 10만 칸짜리 사전 영단어가 늘어선 이 징그러운 초거대 희소 배열(거의 전부 `0`으로 가득 찬 쓰레기)을 그대로 연산하면 서버는 멈춰버립니다.

수학자들은 이 초거대 엑셀 매트릭스 행렬($A$)을 수학의 **특이값 분해(SVD, Singular Value Decomposition)**라는 마법을 써서 **3개의 작은 행렬 조각들의 곱하기($U \times \Sigma \times V^T$)**로 쪼개 버립니다.

## 01. 잠재의미분석 (LSA) 의 절단 마법 (Truncated SVD)
그냥 분해만 하면 의미가 없습니다. LSA의 핵심은 **"자르기(Truncated)"** 에 있습니다.
컴퓨터가 "어차피 저 끝에 달린 조무래기 자잘한 중요도 값(`Sigma`, $\Sigma$) 은 쓰레기 노이즈 관사(`the`, `a`)니까 그냥 칼로 썰어서 쓰레기통에 폐기하자!" 라면서 중요한 상위 `K`개의 축만 남기고 뒤쪽 행렬 파이프를 강제로 절단 절제수술 해버립니다.

![LSA Truncated SVD Theory](./img/../sec01/img/sec01_05.svg)

$$ A \approx U_k \Sigma_k V_k^T $$

1. 거대한 매트릭스가 싹둑 썰리면서, 차원 벡터가 겨우 `3차원(K=3)` 정도로 납작하게 찌그러져 강제 압축됩니다.
2. 기계는 이제 수십만 칸을 스캔할 필요 없이 `[0, 3, 1]` 같은 농축된 3칸짜리 실수 벡터 수치만 보고 문서를 이해하게 됩니다!
3. 학자들은 이 **살아남은 `K`개의 차원 축**이야말로, 단순 단어 스펠링을 넘어선 기저에 깔려있던 **"숨겨진 잠재 토픽(Latent Topic)"** 이라고 억지 비유를 섞어 찬양하게 됩니다.

## 02. 잠재의미분석(LSA)의 치명적인 종말
수학적으로 선형대수학의 궁극을 찍은 LSA 모델은 성능이 제법 괜찮았지만 실무(현업) 서비스에 적용하기엔 너무나 끔찍한 구조적 결함을 안고 있었습니다.

![LSA Limit Problem](./img/../sec01/img/sec01_06.svg)

1.  **눈먼 해석 (이름표 없는 토픽 축의 슬픔)**
    추출된 차원 압축 축(토픽 1번 축)을 사람이 쳐다봐도 이게 도대체 "경제 축인지, 동물 축인지" 명확하게 떨어지지 않습니다. 단어들이 여기저기 애매한 실수 값으로 난잡하게 혼재된 구름 수치 상태로 표출되기 때문에 인간이 "이건 정치 토픽이야!" 라고 명명하는 룰룰루 라벨링 작업조차 매우 까다롭습니다.
2.  **재부팅의 저주 폭발 (새로운 데이터 수용 불가)**
    이게 가장 큰 죄악입니다. LSA는 이미 주어진 거대한 행렬 매트릭스를 처음부터 끝까지 완전체로 쪼개는 방식입니다.
    만약 네이버 뉴스 서버에 오늘 방금 찍힌 초짜 **최신 뉴스 문장 단 한 줄** 이 새로 입력으로 들어오면 어떻게 될까요?
    $\to$ 컴퓨터는 **그 1장을 처리하기 위해 과거 수천만 장의 엑셀 데이터 매트릭스 전체를 밑바닥부터 다시 로드해서 전체 SVD 분해 공식을 처음부터 끝까지 피눈물 나게 다시 다 계산해야 합니다!(실시간 업데이트 불가능)**

이 참혹한 실시간 연산 불가 연쇄 붕괴 에러를 타개하기 위해, 학자들은 매트릭스 행렬 구조에 집착하던 수학책을 완전히 찢어버리고, 아예 철학적 근본(패러다임) 세계관을 뒤엎어버린 **'생성 모델(Generative Model)'**이라는 신대륙으로 배를 돌리게 됩니다.
