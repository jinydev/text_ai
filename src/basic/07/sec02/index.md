# 7.2 잠재축의 파괴적 차원 압축: 잠재의미분석 (LSA)

단순히 K-means 처럼 야생의 문서들을 물리적 거리로 묶어버리는 1차원적 수준을 넘어, 무식하게 거대한 100만 줄짜리 엑셀 단어 행렬(DTM)의 차원을 **우주 프레스기처럼 강제로 찍어 누르며 압축하는 선형 대수학 기법**을 배웁니다. 행렬 쪼개기 기술을 통해 노이즈를 다 박살 내고 숨은 토픽 에센스 축을 캐내는 초기 LSA(Latent Semantic Analysis) 모형입니다.

---

## 7.2.1 차원의 저주와 메스: 특이값 분해 (SVD)

과거 3주 차에 우리는 원시 텍스트를 DTM(문서-단어 행렬) 또는 TF-IDF 형태의 거친 엑셀표로 만들었습니다. 
예를 들어 `세로 1,000만 줄 (말뭉치 코퍼스 전체 문서) × 가로 10만 칸 (영어 사전 전체 단어)` 이 늘어선 이 징그러운 초거대 희소 배열 방은 사실 내부 데이터의 99.9%가 무의미한 숫자 스위치 `0`으로 가득 찬 쓰레기장이었습니다. 이 희소 매트릭스를 그대로 딥러닝 망에 태우면 램(RAM) 서버는 그 자리에서 폭발하여 멈춰버립니다.

수학자들은 이 초거대 엑셀 매트릭스 원본 행렬($A$)을 **특이값 분해(SVD, Singular Value Decomposition)**라는 전설의 선형 대수 곱셈 마법을 써서 다음과 같이 **3개의 작은 행렬 조각들의 곱하기($U \times \Sigma \times V^T$)** 세트로 갈가리 찢어 분해해 버립니다.

$$ A = U \Sigma V^T $$

---

## 7.2.2 LSA의 진짜 칼날: 대절단 마법 (Truncated SVD)

그냥 수식 뼈대대로 분해만 해서는 크기만 똑같아서 아무런 메모리 압축 의미가 없습니다. LSA 수학의 핵심은 컴퓨터가 **"강제로 절단해 버리기(Truncated)"** 에 있습니다.

![LSA SVD Truncation Matrix Math Visualization](./img/svd_math_breakdown.svg)

가장 가운데 들어간 $\Sigma$ (시그마) 대각 행렬은 **"이 데이터 우주공간에서 각 축이 얼마나 파워(영향력)를 갖는지 큰 순서대로 내림차순 정렬된 점수표"** 입니다. 컴퓨터는 여기서 묘수를 씁니다.

1.  **쓰레기 축 버리기**: *"어차피 이 $\Sigma$ 행렬의 저 뒷단 꼬리에 달린 조무래기 9만 개의 자잘한 영향력 점수 벡터들은 보나 마나 `the`, `a` 같은 쓰레기 노이즈 관사들일 거잖아? 그냥 가위로 싹둑 다 썰어서 영원히 쓰레기통에 폐기해버리자!"*
2.  **K 축 압축 ($Truncation$)**: 그래서 가장 덩치 파워가 큰 상위 `K`개(예: 3개~10개)의 특이값 대각 성분만 살려두고 양옆 행렬 파이프를 강제로 절제 수술(자르기) 해버립니다.

$$ A \approx U_k \Sigma_k V_k^T $$

1. 거대한 매트릭스가 싹둑 썰리면서, 10만 개짜리 텍스트 배열 차원 벡터가 **겨우 `3차원(K=3)` 크기의 미니 행렬**로 기적처럼 납작하게 찌그러져 강제 압축됩니다.
2. 기계는 이제 수십만 칸을 뒤져볼 필요 없이, 각 문서를 `[0.45, -2.13, 0.99]` 같은 고도로 농축된 3칸짜리 실수 벡터 수치 조합만 스캔해서 서로의 거리를 비교하면 끝납니다!
3. 통계학자들은 이 참혹한 톱질에서 끝까지 **살아남은 소수의 `K`개의 차원 축**이야말로, 단순 단어 스펠링들을 초월하여 기저에 은밀하게 깔려있던 **"숨겨진 잠재 토픽(Latent Topic)" 파벌** 이라고 칭송하게 됩니다.

## 7.2.3 치명적인 약점: LSA의 새문서 재부팅 저주 폭발

수학적으로 선형대수학 행렬 자르기의 궁극을 찍은 LSA 모델은 성능과 검색 매칭률이 제법 탁월했지만, 실제 네이버 같은 실시간 B2B 포털(현업) 서비스에 상용 도입하기엔 너무나 끔찍한 구조적 결함을 끌어안고 있었습니다.

![LSA Reboot Curse on New Document Addition](./img/lsa_reboot_curse.svg)

1.  **눈먼 해석 (이름표 없는 토픽 축의 슬픔)**
    추출된 차원 압축 축(토픽 1번 축)을 사람이 두 눈으로 쳐다봐도, 축 안에 융합된 숫자가 도대체 "이게 경제 토픽 축인지, 동물 축인지" 명확하게 떨어지지 않습니다. 단어들이 선형 공간 여기저기 애매한 실수 음수 값으로 난잡하게 혼재된 짬뽕 수치 상태로 던져지기 때문에 인간이 역으로 모니터링하며 "이건 정치 토픽이야!" 라고 명명하는 사후 라벨링 작업조차 매우 까다롭습니다.
2.  **🚨 업데이트 불능 재부팅의 멸망 (새 문서의 추가 불가)**
    이게 LSA의 가장 거대하고 악랄한 죄악입니다. LSA 알고리즘 구조는 이미 완성되어 주어진 거대한 엑셀 행렬 매트릭스($A$) 전체를 처음부터 한 덩어리로 통째로 쪼개는 방식입니다.
    만약 네이버 뉴스 서버에 방금 1초 전 발행된 **최신 속보 기사 문장 딱 단 1줄(Document + 1)** 이 새로 입력으로 들어오면 시스템에 무슨 일이 일어날까요?
    $\to$ 컴퓨터는 **그 새로운 "단 1장" 의 트렌드를 기존 행렬에 통합 적용하기 위해, 서버를 멈추고 과거 수천만 장의 엑셀 데이터 매트릭스 전체의 틀을 리셋한 뒤 밑바닥부터 다시 SVD 분해 공식을 처음부터 끝까지 피눈물 나게 다시 다 일괄 재계산해야 합니다! (실시간 업데이트 절대로 불가능)**

이 참혹한 실시간 수학 연산 불가 연쇄 붕괴 에러를 타개하기 위해, 학자들은 기존의 "행렬을 부수자"는 K-means 와 선형대수 매트릭스 수학책을 완전히 불태워 찢어버리게 됩니다. 그리고 데이터 마이닝의 아예 철학적 근본(패러다임) 세계관 자체를 뒤엎어버린 **'생성 확률 통계 모델(Generative Model)'**이라는 황금의 언덕 신대륙으로 배를 돌리게 됩니다. 다음 장에서 그 위대한 서막이 열립니다.
